### Agent System:
- When -m and -a are both supplied, the script scaffolds a Modelfile at
  ~/.config/ollama-fs/agents/<agent_name>/Modelfile (if absent), then runs
  `ollama create` to register it with Ollama.
- The Modelfile template includes FROM (injected from -m at scaffold time),
  SYSTEM prompt, and PARAMETER blocks (temperature, num_ctx, repeat_penalty,
  top_p, top_k).
- A .built sentinel file prevents unnecessary rebuilds — the agent is only
  re-created when the Modelfile has been modified since the last build.
- If -a is supplied without -m, the script assumes the agent is already
  registered in Ollama and skips scaffolding.

### Startup Sequence:
- Agent scaffolding and `ollama create` run before the REPL opens.
- A startup ping fires a non-streaming probe to verify the chat target responds
  before entering the interactive loop. A spinner runs during the ping with the
  message "may take a minute". On failure, a diagnostic checklist is printed
  and the process exits cleanly.
